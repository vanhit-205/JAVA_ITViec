Nguyên tắc triển khai (Rules)•Ngôn ngữ: Java Android.•Kiến trúc: MVVM (Model-View-ViewModel).•View Binding: Luôn sử dụng ViewBinding, không dùng findViewById.•API: Sử dụng Retrofit2 với Interface định nghĩa sẵn.•Phân quyền: Kiểm tra role từ SharedPreferences để ẩn/hiện UI tương ứng.2. Lộ trình thực hiện (Step-by-Step)Bước 1: Thiết lập cấu trúc nền tảng (Base & Utils)AI cần tạo trước các class dùng chung để đồng bộ toàn dự án:1.Constants: Lưu các hằng số API URL, SharedPref keys (TOKEN, ROLE, USER_ID).2.SharedPrefManager: Class quản lý lưu/xóa Token và UserInfo.3.ApiClient: Cấu hình Retrofit với AuthInterceptor để tự động đính kèm Bearer Token.4.BaseResponse: Model chung cho các phản hồi từ API (status, message, data).Bước 2: Luồng Authentication (Usecase 6.1 -> 6.4)Triển khai theo thứ tự:1.LoginActivity:◦Input: Email, Password.◦Logic: Gọi /auth/login -> Lưu Token & Role -> Điều hướng theo Role.2.RegisterActivity:◦Validate: Khớp mật khẩu, định dạng email.3.ForgotPassword Flow:◦Tạo 3 Activity: ForgotPassword (nhập email) -> OtpVerification -> ResetPassword.Bước 3: Luồng Main Navigation (Usecase 7 -> 9)Xây dựng khung ứng dụng chính:1.MainActivity (User): Gồm Bottom Navigation (Home, Jobs, Companies, Profile).2.HomeFragment:◦Hiển thị 3 RecyclerView: Job nổi bật, Company nổi bật, Skill Chips.3.JobDetailActivity:◦Hiển thị chi tiết Job.◦Logic nút "Apply Now": Nếu role là USER thì hiện, nếu là ADMIN/HR thì ẩn hoặc đổi chức năng.Bước 4: Luồng Resume & Application (Usecase 10)Đây là phần quan trọng nhất về logic:1.File Picker: Logic chọn file PDF/Docx từ điện thoại.2.Upload Logic: Sử dụng MultipartBody.Part trong Retrofit để gửi CV lên /resumes.3.MyResumeActivity (User): Hiển thị danh sách Job đã ứng tuyển và trạng thái (PENDING/APPROVED...).4.ResumeManagement (HR):◦Hiển thị danh sách ứng viên.◦Nút bấm chuyển trạng thái: "Reviewing", "Approve", "Reject".Bước 5: Luồng Quản trị (Admin/HR Management - Usecase 12 -> 13)Dành cho người dùng có role cao:1.Job/Company/Skill Management:◦Sử dụng chung một template UI: Danh sách (RecyclerView) + Nút Add (FloatingActionButton) + Menu context (Edit/Delete).2.User Management:◦Chỉ ADMIN truy cập.◦Chức năng: Thay đổi Role hoặc Khóa tài khoản.
